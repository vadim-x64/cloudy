Cloudy 🌤️
Застосунок Cloudy - це розумне метеорологічне рішення нового покоління, розроблене на базі Flutter. 
Проєкт поєднує максимально точні погодні дані, динамічний процедурний рендеринг візуальних ефектів 
та інтегрованого ШІ-асистента для надання персоналізованих порад.

✨ Ключові можливості
Детальна метеорологія: Отримання поточних даних про температуру, відчуття температури, вологість, 
тиск, швидкість вітру, індекс якості повітря (AQI), а також формування графіка на 24 години та 
прогнозу на 5 днів.
Інтерактивний ШІ-Асистент: Вбудований розумний чат на базі хмарних LLM (Groq або X.ai), який 
автоматично отримує системний контекст із поточними метеоданими для точних порад (наприклад, щодо 
вибору одягу чи парасолі).
Процедурний рендеринг і графіка: Математична генерація ефектів дощу, снігу, туману, блискавок та 
полярного сяйва на GPU за допомогою CustomPainter та тригонометричних функцій часу, що дозволяє 
уникати використання важких растрових зображень.
Динамічний UI: Використання ефектів матового скла (через розмиття Гауса) та розумних градієнтів із 
Lottie-анімаціями, які плавно адаптуються до часу доби (світанок, ранок, день, вечір, ніч, сутінки) 
та погодних умов.
Глобальна геокартографія: Карта на базі flutter_map із тайловими шарами OpenWeather (опади, 
хмарність, температура, вітер) та живим моніторингом катаклізмів — координати пожеж від NASA EONET 
та землетрусів від USGS парсяться в інтерактивні маркери.
Розумна локація: Двохетапна ідентифікація через високоточний GPS-модуль (geolocator) з автоматичним 
перемиканням на ручний пошук міст (автокомпліт через OpenWeather API) у разі відсутності дозволів 
на геолокацію.

🏗️ Архітектура системи
Потік даних (Data Pipeline): Реалізовано реактивну обробку даних із багатопотоковими запитами 
(Future.wait) для паралельного завантаження поточної погоди, 5-денного прогнозу та AQI, що скорочує 
час очікування відповіді мережі на 60%.
Збереження сесії та стану: Використання асинхронного сховища SharedPreferences для кешування 
параметрів користувача (наприклад, одиниць виміру) та циклічного буфера повідомлень для збереження 
контексту діалогу із ШІ.
Clean Architecture: Проєкт розроблено з дотриманням принципів чистої архітектури, що гарантує 
надійність, легкість підтримки та масштабованість коду.

🚀 Встановлення та запуск
Попередні вимоги
Для правильної роботи програми необхідно налаштувати змінні середовища. Створіть файл .env у 
кореневій директорії проєкту та додайте наступні ключі доступу:
OPENWEATHER_API_KEY=ваш_ключ_від_openweather
GROK_API_KEY=ваш_ключ_від_grok_або_groq
AI_GREETING_PROMPT="Привіт ШІ!"
AI_CHAT_PROMPT="Ти - професійний ШІ-помічник у додатку..."

Запуск проєкту
Клонуйте репозиторій:
git clone [https://github.com/ваш_юзернейм/cloudy.git](https://github.com/ваш_юзернейм/cloudy.git)
cd cloudy

Встановіть залежності:
flutter pub get

Запустіть застосунок:
flutter run
