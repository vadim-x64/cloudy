# 🌦️ Cloudy

**Cloudy** - це динамічний Flutter-застосунок погоди для Android, який поєднує класичний 
метеомоніторинг із візуально насиченим інтерфейсом, адаптивними фонами, прогнозами, картою, 
геолокацією, перекладом назв, а також вбудованим AI-асистентом.

Проєкт створено як “живий” погодний інструмент: він не просто показує температуру, а підлаштовує 
вигляд під час доби та погодні умови, дає короткі пояснення, показує 24-годинний і 5-денний 
прогноз, інформує про якість повітря, підтримує ручний пошук міст і автоматичне визначення локації.

## Можливості

### 1. Поточна погода
- температура;
- “відчувається як”;
- опис стану погоди;
- вологість;
- швидкість вітру;
- опади;
- атмосферний тиск;
- якість повітря (AQI).

### 2. Прогнози
- прогноз на **24 години**;
- прогноз на **5 днів**;
- агрегування денних мінімумів і максимумів;
- підбір найрелевантнішої погодної іконки для дня/години.

### 3. Геолокація та пошук
- автоматичне визначення локації через GPS;
- ручний пошук міста через автодоповнення;
- підтримка повторного оновлення даних для поточної або вибраної локації.

### 4. Адаптивний інтерфейс
- градієнтний фон, який змінюється залежно від:
    - часу доби;
    - поточної погоди;
- Lottie-анімації для різних погодних станів;
- кастомні анімовані погодні іконки;
- glassmorphism-стилізація через blur-ефекти;
- анімації появи, переходів і плавних станів UI.

### 5. AI-асистент
- чат із погодним контекстом;
- персоналізовані поради щодо одягу, парасолі, комфорту;
- збереження історії діалогу в межах сесії;
- підтримка різних моделей через Groq або x.ai;
- промпти налаштовуються через `.env`.

### 6. Карта погоди
- інтерактивна карта на базі FlutterMap + OpenStreetMap;
- шари погоди OpenWeatherMap:
    - опади,
    - хмарність,
    - температура,
    - вітер,
    - тиск;
- додаткові маркери подій/катаклізмів;
- тап по карті для отримання погоди конкретної точки.

### 7. Налаштування та зручність
- перемикання між **°C / °F / K**;
- автоскидання/оновлення стану;
- інформація про застосунок;
- показ версії та build number;
- підказки для першого запуску.

## Як це працює

Cloudy побудований як реактивний погодний клієнт.

### Потік даних
1. Застосунок стартує та завантажує конфігурацію з `.env`.
2. Отримує координати через GPS або бере місто з пошуку.
3. Паралельно запитує:
    - поточну погоду;
    - прогноз;
    - дані AQI.
4. Парсить відповідь у модель `WeatherModel`.
5. На основі:
    - `mainCondition`,
    - `iconCode`,
    - `sunrise/sunset`,
    - `timezoneOffset`,
    - поточної години
      формується:
    - фон,
    - анімації,
    - частина доби,
    - текстові підказки,
    - візуальні елементи.

### Чому так зроблено
Такий підхід дозволяє:
- уникати “сухого” UI;
- зробити інтерфейс залежним від реального стану погоди;
- швидше завантажувати дані за рахунок паралельних запитів;
- підтримувати як автоматичну, так і ручну навігацію по містах.

## Архітектура проєкту

Архітектура проєкту розділена на логічні шари:

### `models`
Містить структури даних:
- `WeatherModel`
- `HourlyForecast`
- `DailyForecast`
- `CitySuggestion`
- `LocationException`

### `services`
Відповідає за інтеграцію із зовнішніми джерелами:
- `WeatherService` - OpenWeather API, переклад назв, прогноз, AQI;
- `LocationService` - доступ до геолокації через Geolocator;
- `AiService` - AI-чат через Groq або x.ai.

### `screens`
Візуальні екрани застосунку:
- `HomeScreen`
- `AppInfoScreen`
- `SplashScreen` та інші допоміжні екрани/переходи.

### `widgets`
Кастомні інтерфейсні елементи:
- анімовані погодні іконки;
- графік температури;
- оверлеї погоди;
- завантажувач;
- модалка AI-чату;
- модалка карти;
- tutorial overlay.

## Стек технологій

- **Flutter / Dart** - основа застосунку.
- **OpenWeatherMap API** - погода, прогноз, AQI, погодні шари.
- **Geolocator** - GPS і визначення поточної локації.
- **translator** - переклад назв міст і регіонів на українську.
- **flutter_dotenv** - зчитування ключів API та prompt-налаштувань.
- **shared_preferences** - збереження першого запуску та локальних налаштувань.
- **lottie** - анімації фону.
- **package_info_plus** - версія застосунку.
- **FlutterMap / OpenStreetMap** - інтерактивна карта.
- **HTTP** - мережеві запити.
- **Google Fonts** - типографіка.

## Структура даних

### `WeatherModel`
Основна модель, яка містить:
- назву міста, регіон, країну;
- температуру;
- відчуття;
- опис;
- іконку;
- головний погодний стан;
- вологість;
- швидкість вітру;
- опади;
- тиск;
- AQI;
- локальний час;
- часовий пояс;
- схід і захід сонця;
- погодинний прогноз;
- денний прогноз.

### Логіка визначення часу доби
Модель визначає:
- **Ніч**
- **Світанок**
- **Ранок**
- **День**
- **Полудень**
- **Вечір**
- **Сутінки**

Це потрібно не тільки для текстового відображення, а й для вибору:
- градієнта фону;
- Lottie-анімації;
- поведінки погодної графіки;
- візуального стилю.

### Підбір погодної іконки
У прогнозах враховується пріоритет погодних кодів, щоб при змішаній відповіді система показувала 
найбільш значущий стан, а не перший випадковий код.

### Попередні вимоги
- Flutter SDK;
- Android Studio / VS Code;
- Android device або emulator;
- API-ключі для OpenWeatherMap;

```bash
git clone https://github.com/your-username/cloudy.git
cd cloudy
flutter pub get
flutter run
```

## Налаштування `.env`

У корені проєкту потрібно створити файл `.env`:

```env
OPENWEATHER_API_KEY=your_openweather_api_key
GROK_API_KEY=your_groq_or_xai_api_key
AI_GREETING_PROMPT=your_greeting_prompt
AI_CHAT_PROMPT=your_prompt
```

### Що робить кожна змінна

- **`OPENWEATHER_API_KEY`** - доступ до погодних, прогнозних і AQI-ендпоінтів.
- **`GROK_API_KEY`** - доступ до AI-чату.
- **`AI_GREETING_PROMPT`** - системний prompt для першого вітального повідомлення.
- **`AI_CHAT_PROMPT`** - системний prompt для погодного чату з контекстом.

### Важливо
Не коміть `.env` у репозиторій. У проєкті він має бути локальним.

## Права доступу Android

У `AndroidManifest.xml` уже закладено необхідні дозволи:

- `INTERNET`
- `ACCESS_FINE_LOCATION`
- `ACCESS_COARSE_LOCATION`

Це потрібно для:
- завантаження API-даних;
- отримання GPS-координат;
- роботи пошуку та карти.

## Основні сценарії використання

### Автоматичний сценарій
1. Користувач відкриває застосунок.
2. Система запитує локацію.
3. Показує погоду для поточної точки.
4. Підтягує фон, іконку, прогноз і додаткові дані.

### Ручний сценарій
1. Користувач вводить назву міста.
2. Отримує підказки автодоповнення.
3. Обирає потрібне місто.
4. Бачить погоду саме для цієї локації.

### Поглиблений перегляд
- натискання на температуру запускає повноекранну погодну анімацію;
- кнопка деталей відкриває розширені дані;
- кнопка карти відкриває погодну карту;
- кнопка AI відкриває чат із погодним контекстом;
- екран “Про застосунок” пояснює логіку та стек.

## Обробка помилок

Проєкт обробляє типові збої окремо:

### Геолокація
- GPS вимкнено;
- дозвіл відхилено;
- дозвіл відхилено назавжди.

У таких випадках користувач отримує зрозумілий діалог із рекомендацією:
- увімкнути локацію;
- відкрити налаштування застосунку;
- перейти на ручний пошук.

### Мережа
Є окремі повідомлення для:
- відсутності інтернету;
- слабкого сигналу;
- обриву з’єднання під час запиту.

### AI
Якщо ключ недоступний або модель не відповіла:
- чат не падає;
- користувач отримує запасне повідомлення;
- застосунок продовжує працювати.

## Додаткові примітки

### Локалізація
Назви міст, регіонів і країн переводяться на українську, щоб інтерфейс був природнішим для користувача.

### Оновлення даних
Застосунок періодично оновлює стан і також перезавантажує погоду при поверненні у foreground.

### UX-рішення
У проєкті свідомо використано:
- прозорі панелі;
- великий візуальний акцент на температурі;
- анімації появи;
- контекстні фони;
- зрозумілі підказки для першого запуску.

Це робить Cloudy не просто утилітою, а повноцінним атмосферним інтерфейсом.
